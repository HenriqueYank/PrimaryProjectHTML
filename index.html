O processo de usar entidades HTML para exibir caracteres especiais, como <e>, em vez de interpretá-los como tags HTML, é conhecido como "escapamento de caracteres" ou "escapamento de HTML". Este processo é necessário para que o navegador mostre os caracteres literais em vez de tratá-los como parte da sintaxe HTML.
Aqui estão alguns exemplos de entidades HTML comuns usadas.

  • < é representado por &lt; (less than)
  • > é representado por &gt; (greater than)
  • & é representado por &amp; (ampersand)
  • " é representado por &quot; (double quote)
  • ' é representado por &apos; (single quote)

Tag: <h1> até <h6>

( Voltar )

As tags <h1> até <h6> são usadas para definir cabeçalhos (ou títulos) de diferentes níveis em uma página web. Essas tags ajudam a estruturar o conteúdo, criando uma hierarquia visual e semântica.

<h1> define o título mais importante. <h6> define o título menos importante.

Nota: Use apenas um <h1> por página - deve representar o título/assunto principal de toda a página. Além disso, não pule os níveis de título - comece com <h1> , depois use <h2> e assim por diante.

Tag : <p>

( Voltar )

A tag <p> é essencial para a estruturação de texto em HTML, permitindo que você crie parágrafos claramente definidos e estilizados. Usando CSS, você pode ajustar a aparência dos parágrafos conforme necessário para criar uma experiência de leitura agradável e visualmente atraente em sua página web.

Características da Tag <p>

  1. Bloco de Elemento:
    • A tag <p> é um elemento de bloco, o que significa que ela ocupa toda a largura disponível e começa em uma nova linha, separando seu conteúdo dos elementos anteriores e posteriores.

  2. Margem Padrão:
    • Os navegadores aplicam margens padrão aos parágrafos para criar espaçamento vertical entre eles. Você pode ajustar essas margens usando CSS.

Importância Semântica

Além de estilizar e organizar o texto, o uso correto da tag <p> é importante para a acessibilidade e a semântica do conteúdo. Os leitores de tela e os motores de busca interpretam as tags <p> como parágrafos de texto, o que ajuda na navegação e na indexação do conteúdo.

Considerações

  • Acessibilidade: Usar a tag <mark> pode ajudar na acessibilidade, destacando informações importantes que precisam ser facilmente localizadas.
  • SEO: O uso da tag <mark> pode ajudar os mecanismos de busca a entender quais partes do texto são importantes, potencialmente melhorando a relevância do conteúdo para determinadas consultas.
  • Usabilidade: Destacar partes do texto pode melhorar a usabilidade da página, guiando o usuário para informações importantes de maneira rápida e eficiente.

A tag <mark> é uma ferramenta útil para destacar texto de maneira semântica, clara e visualmente distinta em uma página web.

Tag : <strong>

( Voltar )

A tag <strong> em HTML é usada para indicar que o texto é de grande importância, e geralmente é renderizada em negrito pelos navegadores. Semânticamente, a tag <strong> adiciona ênfase forte ao texto, sugerindo que ele é mais significativo ou urgente do que o texto ao redor.

Este é um texto de exemplo fortemente enfatizado.

Considerações

  • Acessibilidade: A tag <strong> não apenas aplica negrito ao texto, mas também informa leitores de tela sobre a ênfase, tornando-a importante para acessibilidade.
  • SEO: O uso apropriado de <strong> pode ajudar os motores de busca a entender a hierarquia e a importância do conteúdo na página.

Diferença entre <strong> e <b>

  • <strong>: Usada para dar ênfase forte e tem significado semântico
  • <b>: Usada apenas para estilização visual, aplicando negrito sem implicações semânticas.

Tag : <i>

( Voltar )

A tag <i> em HTML é usada para renderizar texto em itálico. Semânticamente, a tag <i> pode ser usada para indicar texto que necessita de ênfase leve, termos técnicos, títulos de obras, pensamentos ou diálogos internos, entre outros. No entanto, para ênfase forte, é mais apropriado usar a tag <em>.

Este é um exemplo de itálico.

Considerações

  • Acessibilidade: A tag <i> é geralmente usada para estilizar texto, mas não fornece a mesma ênfase semântica que a tag <em>. Use <em> para ênfase que deve ser interpretada por leitores de tela.
  • SEO: A tag <i> não tem um impacto significativo no SEO, mas o uso correto de tags semânticas pode contribuir para uma melhor acessibilidade e, indiretamente, para a experiência do usuário.
  • Usabilidade: Usar a tag <i> ajuda a estilizar texto de forma que certas palavras ou frases se destacam visualmente, tornando o conteúdo mais fácil de ler e entender.

Alternativas Semânticas

Para ênfase semântica, é recomendável usar a tag <em>. A tag <i> deve ser usada quando a intenção é apenas estilística.

Semânticamente, pode ser utilizada para destacar texto especial, indicar termos estrangeiros, nomes de obras ou textos que precisam ser enfatizados de maneira diferente do itálico ou negrito. Em algumas práticas, também pode ser usada para indicar erros ortográficos ou correções.

Este é um exemplo de sublinhado.

  • Acessibilidade: O uso da tag <u> deve ser feito com cautela, pois a sublinha geralmente indica links para muitos usuários. Certifique-se de que o texto sublinhado não seja confundido com um link.
  • Usabilidade: Sublinhado pode ajudar a destacar termos especiais, mas deve ser usado moderidamente para evitar confusão com links.

Alternativas Semânticas

Para ênfase semântica diferente, considere o uso de outras tags como <em> ou <strong>, que são mais apropriadas para ênfase de leitura por leitores de tela.

Tag : <ol> - Lista Ordenada

( Voltar )

A tag <ol> em HTML é usada para criar listas ordenadas, onde os itens são numerados automaticamente. Esta tag é frequentemente usada quando a ordem dos itens é significativa, como em instruções passo a passo, listas de tarefas, ou classificações.

Atributos Comuns

Usado dentro da tag <ol>.

  • Type: Define o tipo de marcador para os itens da lista (1, A, a, I, i).
  • Start: Define o número inicial da lista.
  • Reversed : Reverte a ordem da lista, começando do número mais alto para o mais baixo.

Exemplo Lista Ordenada

  1. Item 1
  2. Item 2
  3. Item 3

Considerações

  • Semântica: A tag <ol> deve ser usada quando a ordem dos itens é importante.
  • Acessibilidade: Ajuda leitores de tela a entender a importância e a sequência dos itens listados.
  • Usabilidade: Facilita a leitura e a compreensão de listas sequenciais.

Tag : <ul> - Lista Não Ordenada

( Voltar )

A tag <ul> em HTML é usada para criar listas não ordenadas, onde os itens da lista são precedidos por marcadores (bullets) em vez de números. Essa tag é ideal para listas onde a ordem dos itens não é significativa.

Atributos Comuns

Usado dentro da tag <ul> o atributo type = (+) o tipo do atributo.

  • type : Define o tipo de marcador para os itens da lista. Os valores possíveis incluem:
    • disc: (padrão, círculo preenchido)
    • circle: (círculo vazio)
    • square : (quadrado)

Exemplo Lista Não Ordenada

  • Item A
  • Item B
  • Item C

Considerações

  • Semântica: Use a tag <ul> quando a ordem dos itens não é importante.
  • Acessibilidade: As listas não ordenadas são facilmente compreendidas por leitores de tela, facilitando a navegação e a compreensão do conteúdo.
  • Usabilidade: Listas não ordenadas são úteis para agrupar itens relacionados sem sugerir uma hierarquia ou sequência específica.

Tag : <li>

( Voltar )

A tag <li> em HTML é usada para definir itens em uma lista, seja ela ordenada (<ol>) ou não ordenada (<ul>). Cada item de lista deve ser colocado dentro de uma tag <li> para ser corretamente interpretado como parte da lista.

Considerações

  • Semântica: A tag <li> deve ser usada dentro de <ul> ou <ol> para definir itens de lista. Ela é essencial para que os navegadores e leitores de tela interpretem corretamente a lista.
  • Acessibilidade: Usar <li> dentro das tags de lista apropriadas melhora a acessibilidade, permitindo que leitores de tela anunciem corretamente a estrutura da lista.
  • Flexibilidade: Itens de lista podem conter quase qualquer conteúdo, incluindo texto, imagens, links, e até outras listas.

Tag : <a>

( Voltar )

A tag <a> em HTML é usada para criar hiperlinks, também conhecidos como âncoras ou links de navegação. Esses links permitem que os usuários cliquem em um texto ou imagem para navegar para outra página da web, um arquivo, uma seção da mesma página ou executar uma ação.

Este é um exemplo de um link

Atributos

  • href: Atributo obrigatório que especifica o destino do link. Pode ser uma URL (endereço web), um caminho de arquivo, um ID de elemento (para rolar para uma seção na mesma página), ou um protocolo como mailto* : para e-mails.
    mailto* é um protocolo em links HTML que abre o cliente de e-mail do usuário para enviar um novo e-mail ao endereço especificado.
  • texto do link: O texto ou imagem que os usuários veem e clicam para seguir o link.

Atributos opcionais

  • target: Especifica onde abrir o destino do link. Os valores comuns incluem _blank para abrir em uma nova janela ou guia, _self para abrir na mesma janela, _parent para abrir no quadro pai, e _top para abrir na janela de nível superior.
  • title: Fornece um título para o link, geralmente exibido como uma dica de ferramenta quando o usuário passa o mouse sobre o link.

Considerações

  • Semântica: Use <a> para links e botões de navegação, não apenas para acionar ações com JavaScript.
  • Acessibilidade: Links devem ser claramente distinguíveis e indicar claramente seu destino.
  • Compatibilidade: Verifique se os links funcionam corretamente em diferentes navegadores e dispositivos.

Tag : <hr>

( Voltar )

A tag <hr> é usada para inserir uma linha horizontal, geralmente usada para separar seções de conteúdo. Tradicionalmente, ela era usada para denotar uma mudança temática entre parágrafos de texto, mas atualmente pode ser estilizada e utilizada para vários propósitos visuais.
A tag <hr> é uma tag de fechamento automático, ou seja, não precisa de uma tag de fechamento

Atributos Comuns

  • aling: Define o alinhamento da linha. Valores possíveis são left, center, e right.
  • color: Define a cor da linha.
  • size: Define a espessura da linha.
  • width: Define a largura da linha.

* Todos esses a atributos são obsoletos e deve ser substituído por CSS.

Considerações

  • Semântica: A tag <hr> pode ser usada para representar uma mudança temática ou separação de conteúdo dentro de uma página.
  • acessibilidade: Quando usada adequadamente, a tag <hr> pode melhorar a acessibilidade ao fornecer uma clara separação visual e semântica entre diferentes partes do conteúdo.
  • Estilização: Use CSS para estilizar a linha horizontal de acordo com o design da sua página, em vez de depender dos atributos obsoletos da tag.

Tag : <font>

( Voltar )

A tag <font> em HTML é usada para definir a aparência do texto, incluindo a cor, a família da fonte e o tamanho do texto. No entanto, é importante notar que a tag <font> é obsoleta no HTML5, e o uso de CSS é recomendado para estilizar o texto.

Atributos comuns

  • color: Define a cor do texto.
  • face: Define a família da fonte do texto.
  • size: Define o tamanho do texto. Os valores podem variar de 1 a 7, sendo 3 o tamanho padrão.

Considerações

  • Obsolescência: A tag <font> é obsoleta no HTML5. O uso de CSS para estilizar o texto é fortemente recomendado.
  • Semântica: Usar CSS para estilização separa o conteúdo da apresentação, mantendo o HTML semântico e mais fácil de manter.
  • Compatibilidade: Embora a tag <font> ainda funcione na maioria dos navegadores, seu uso não é recomendado para novos projetos devido à obsolescência.

A tag <font> foi útil no passado para estilizar texto, mas hoje em dia, a utilização de CSS é a prática padrão e recomendada para obter um design consistente e separar a estrutura do conteúdo da sua apresentação visual.

  • Compatibilidade: A tag <del> é amplamente suportada por todos os navegadores modernos e dispositivos.

Tag : <code>

( Voltar )

A tag <code> em HTML é usada para definir um trecho de código em um documento. O conteúdo dentro da tag <code> é exibido em uma fonte monoespaçada (ou "mono" ou "fixed-width"), o que ajuda a distingui-lo do texto comum. Esta tag é especialmente útil para exibir código-fonte, comandos de terminal, ou qualquer outro tipo de texto que precisa ser mostrado em um formato específico.

Alguns exemplos de comandos em diferentes linguagens de programação:

  • HTML: <p>Olá, mundo!</p>

Considerações

  • Semântica: A tag <code> é semanticamente apropriada para marcar trechos de código, ajudando a diferenciar o código do texto comum.
  • Acessibilidade: O uso adequado da tag <code> pode melhorar a acessibilidade, proporcionando uma indicação clara de que o texto é um trecho de código.
  • Compatibilidade: A tag <code> é amplamente suportada por todos os navegadores modernos e dispositivos.

Tag : <br>

( Voltar )

A tag <br> em HTML é usada para inserir uma quebra de linha, ou seja, ela força o texto a continuar na linha seguinte. Essa tag é uma tag vazia, o que significa que não possui uma tag de fechamento. É útil quando você deseja inserir uma nova linha dentro de um parágrafo ou qualquer outro bloco de texto sem iniciar um novo bloco.

Considerações

  • Semântica: A tag <br> deve ser usada com moderação e apenas para quebras de linha que são parte do conteúdo, não para controle de layout.
  • Acessibilidade: O uso excessivo de <br> para fins de layout pode prejudicar a acessibilidade. Em vez disso, use CSS para controlar o layout e o espaçamento.
  • Compatibilidade: A tag <br> é amplamente suportada por todos os navegadores modernos e dispositivos.
